@switch (step) { @case (0) {

Put publisher records in first

Everything else in this app builds on data you save on Add Records.

  • Search only finds publishers you have already saved for at least one service year.
  • Print and export use those same saved records—there is nothing to open until the record exists.
  • Dashboard totals (publishers, pioneers, elders, and so on) count rows you have stored for the selected year.

Start by choosing the correct service year (next step), then add or copy publishers and fill in monthly ministry information before relying on Search or reports.

} @case (1) {

Service year

The service year starts in September (e.g. 2024 means Sep 2024–Aug 2025). The year selector at the top is shared across Dashboard, Add Records, and Search—set it before you add or edit records so data lands in the right year.

} @case (2) {

Add a publisher

This is where you create the record the rest of the app uses. Open Add Records, enter the publisher’s name (required), optional profile fields, then fill the monthly grid: shared in ministry, Bible studies, auxiliary pioneer, hours, and remarks. Click Save Record.

Saving updates an existing row if the same name already exists for that service year.

} @case (3) {

Start a new year faster

After last year’s records exist, use Copy Previous Year Data on Add Records to bring every publisher’s profile into the new year with blank monthly fields—then complete each month as you go. Names that already exist in the new year are skipped.

} @case (4) {

Search

Once records are saved, use Search or the search box on the Dashboard. Search matches names across all service years. The list is filtered to the selected year—change the year to see the same person in another year.

No matches usually means no record was saved yet for that name—go back to Add Records first.

} @case (5) {

Reports & print

Monthly data you saved is the report for that year. To change it, use Add Records. On Search, expand a publisher and use Print or Export as HTML for an official-style layout.

Printing reflects what is already stored—update the record on Add Records if something is missing or wrong.

} }
